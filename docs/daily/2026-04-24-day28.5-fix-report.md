## 一、真实 API 结构验证结论

通过实际调用 eBay Fulfillment API 和 Finances API（订单 `02-14554-85006`），确认以下字段路径：

### Fulfillment API（`api.ebay.com/sell/fulfillment/v1/`）

| 字段 | 正确路径 | 原错误路径 |
|------|---------|-----------|
| shipping_cost | `pricingSummary.deliveryCost.value` | `fulfillmentHrefs[0].shippingCost` — `fulfillmentHrefs` 对未发货订单为空数组 |
| total_due_seller | `paymentSummary.totalDueSeller.value` | 新增字段 |
| sold_via_ad_campaign | `properties.soldViaAdCampaign` | 新增字段 |
| ebay_fee（fallback） | `totalMarketplaceFee.value` | 原错误读取 `lineItems[].itemTxSummaries[]`（此字段不存在） |

### Finances API（`apiz.ebay.com/sell/finances/v1/transaction` — 路径为 **singular**，不是 `transactions`）

| 字段 | 路径 |
|------|------|
| FEE（主路径） | `transactions[].orderLineItems[].marketplaceFees[].amount` |
| AD_FEE | `transactions[].feeType == "AD_FEE"` → `amount`（留待 Day 31-B） |
| SALE_TAX | 嵌入 SALE transaction 的 `ebayCollectedTaxAmount`（留待 Day 31-B） |
