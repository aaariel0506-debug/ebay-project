## Day 22：线上线下库存联动 ✅

### 核心功能

**`modules/inventory_online/event_handlers.py`**（新建）

`handle_stock_out(event_type, payload)` — STOCK_OUT 事件处理器：

1. **解析 payload**：`sku` / `quantity` / `related_order`
2. **防重复扣减**：查询 `event_log` 中是否有同 `related_order` 的 DONE STOCK_OUT，有则跳过
3. **查询 eBay Listing**：无 listing 或无 `ebay_item_id` 时跳过
4. **计算新库存**：`new_qty = max(current_qty - quantity, 0)`
5. **调用 QuantityAdjuster**：执行 eBay API 扣减
6. **失败传播**：若 `adjust_ebay_quantity` 返回 `success=False`，抛出 `RuntimeError` → `_dispatch` 标记事件 FAILED

### `main.py` 注册

新增 `_register_event_handlers()`，进程启动时一次性注册所有 handler：
```python
bus.subscribe("STOCK_OUT", handle_stock_out)
```

### CLI 链路

```
python main.py inventory offline outbound --sku X --quantity 3 --order ORDER-001
→ outbound() → Inventory(type=OUT) → STOCK_OUT事件发布 → handle_stock_out()
→ QuantityAdjuster → eBay Inventory API 扣减 → LISTING_UPDATED事件
```

## 技术问题与解决

### Bug 1：EbayListing fixture 参数错误
- **错误**：`TypeError: 'price/currency/marketplace_id' is invalid keyword`
- **原因**：测试 fixture 用了不存在的字段名（`price` 应为 `listing_price`，无 `currency`/`marketplace_id`）
- **修复**：对照 `core/models/listing.py` 修正 fixture

### Bug 2：`publish()` 返回 detached instance
- **错误**：`DetachedInstanceError: Instance <EventLog> is not bound to a Session`
- **原因**：`publish()` 在 session A 创建 EventLog → commit → session A 关闭；返回的 record 在 session B 访问时 detached
- **解决**：测试 helper 不依赖 `publish()` 返回值，改为 `db_session.query(EventLog).filter(id > before_id)` 获取最新事件

### Bug 3：SQLite JSON `.astext` 不支持
- **原因**：`EventLog.payload["related_order"].astext` 在 SQLite 下不支持
- **解决**：改为 Python 层面过滤 `ev.payload.get("related_order") == related_order`

### Bug 4：`adjust_ebay_quantity` 不抛异常
- **原因**：API 错误被内部捕获，返回 `AdjustmentResult(success=False)` 而不抛
- **影响**：handler 内的 `except Exception` 无法捕获
- **修复**：在 `result.success is False` 时显式 `raise RuntimeError(...)`
