## 阻塞问题修复

### 阻塞 #1：test_price_monitor_db.py 3 个测试稳定失败

**错误**：`sqlite3.IntegrityError: NOT NULL constraint failed: supplier_price_history.id`

**根因**：`alembic/versions/102cfcaae272_create_supplier_price_history.py` 中 `id` 列定义为 `sa.Integer()` + `autoincrement=True`，但**缺少 `primary_key=True`**。

SQLite 的 `autoincrement` 机制只对 `INTEGER PRIMARY KEY` 列生效。手写 Alembic 迁移时，`sa.Integer()` 不会自动携带 `PRIMARY KEY` 约束，导致 INSERT 时 id 为 NULL，触发 NOT NULL 约束错误。

Mac 和 Linux 上 SQLAlchemy 行为差异来源于此。

**修复**：
```python
# 修复前
sa.Column('id', sa.Integer(), autoincrement=True, nullable=False)

# 修复后
sa.Column('id', sa.Integer(), primary_key=True, autoincrement=True, nullable=False)
```

**教训**：新 Alembic 迁移优先用 `--autogenerate`，让 SQLAlchemy 从模型定义自动推导列属性，手写容易漏掉 `primary_key=True`。

## 技术教训

**守恒断言 > 具体值断言**

```python
# 任何涉及「聚合值」的改动，加这条永远值
assert sum(item["locations"].values()) == item["available_quantity"]
```

**Alembic 手写迁移的 PK 陷阱**

SQLite autoincrement 只在 `INTEGER PRIMARY KEY` 生效。手写迁移必须显式加 `primary_key=True`，`autoincrement=True` 单独不够。
