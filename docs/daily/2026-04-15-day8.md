# Day 8 — 2026-04-15（续）

## 目标
变体 Listing + 图片上传

## 完成内容

### modules/listing/schemas.py — 扩展
- `VariantSpecific`：单维度（Size/M, Color/Red）
- `VariantItem`：独立 SKU + 价格 + 库存 + 变体规格
- `InventoryItemGroupRequest`：变体组请求（至少 2 个变体）
- `VariantListingCreateResponse`：返回 group_id + 各变体 offer_id/listing_id
- `ImageUploadRequest`：支持 `source_type: file|url`
- `ImageValidationResult` + `ImageUploadResponse`：图片校验结果

### modules/listing/utils.py — 扩展
- `build_inventory_item_group()`：构建 ItemGroup payload，deduplicate variantSpecifics
- `build_variant_payload()`：为单个变体构建 InventoryItem payload（引用 group）
- `validate_image_files()`：本地文件 + URL 校验（格式/大小），返回 valid_urls + validation_results
- `ALLOWED_IMAGE_FORMATS` / `MAX_IMAGE_SIZE_BYTES = 7MB` 常量

### modules/listing/service.py — 扩展
- `create_variant_listing()`：完整多变体流程
  1. 各变体 createOrReplaceInventoryItem
  2. 创建 InventoryItemGroup
  3. 各变体 createOffer
  4. 各变体 publishOffer
  5. DB 写入 + LISTING_CREATED 事件
- `_build_variant_inventory_payload()` / `_build_inventory_item_group_payload()`
- `_create_variant_offer()` / `_publish_variant_offer()`
- `_rollback_inventory_items()`：回滚已创建的 inventory items
- `_save_variant_records()`：批量写入 EbayListing + Product
- `validate_images()`：暴露校验逻辑，支持 file:// URL 和外部 URL

### tests/test_variant_listing.py — 新建（30 tests）
- `TestVariantSchemas`（7 tests）：VariantItem / InventoryItemGroupRequest / 响应
- `TestImageSchemas`（6 tests）：ImageUploadRequest / ImageValidationResult
- `TestVariantUtils`（4 tests）：build_variant_payload / build_inventory_item_group
- `TestImageValidation`（7 tests）：URL 校验 / 扩展名 / 大小 / 不存在 / 混合
- `TestVariantServiceUnit`（2 tests）：完整流程 + Step1 失败回滚
- `TestImageService`（3 tests）：图片校验服务
