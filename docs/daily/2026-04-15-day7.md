## Bug Fix（Day 7 结束后发现并修复）

### 🔴 API 路径缺少前缀（生产必挂）
- **问题**：`service.py` 中 4 处路径均为相对路径（`/inventory_item/{sku}` 等），实际 eBay API 需 `/sell/inventory/v1/` 前缀，真实调用必 404
- **修复**：全部补全为 `/sell/inventory/v1/inventory_item/{sku}` 等 4 处
- **测试未发现原因**：API client 被整体 mock，无法触发真实路径校验

### 🟡 Rate Limiter 路径匹配失效
- **问题**：`rate_limit_config.py` 的 key 格式为 `"POST /sell/inventory/v1/inventory_item"`（带 HTTP 方法），但 `client.py` 调用 `rate_limiter.record(path)` 仅传 path，导致全部匹配不上，限流形同虚设
- **修复**：去掉所有 key 的 `GET/POST/PUT/DELETE` 前缀，改为纯 path 前缀

### 🟡 ProductStatus 未导出
- **问题**：`core/models/__init__.py` 导出了其他 5 个 Status/Type enum，唯独漏了 `ProductStatus`
- **修复**：在 `__init__.py` 中补充导出

### 🟡 ProductStatus 写法不统一
- **问题**：其他 5 个 enum 均用 `enum.StrEnum`，唯独 `ProductStatus` 用旧写法 `(str, enum.Enum)`
- **修复**：改为 `class ProductStatus(enum.StrEnum)`

### 🟡 ebay_rate_limits.yaml 死代码
- **问题**：`rate_limit_config.py` 硬编码 dict，yaml 文件从未被加载，也无任何引用
- **修复**：删除 `config/ebay_rate_limits.yaml`，更新 `rate_limiter.py` 类文档字符串

### 🟡 Rate Limit Config Browse API 残留
- **问题**：`rate_limit_config.py` 中 Browse API 两行（`/buy/browse/v1/`、`/buy/deal/v1/`）仍带 `"GET "` 前缀
- **修复**：统一去掉
