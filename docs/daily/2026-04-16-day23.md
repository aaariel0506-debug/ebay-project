## Day 23：库存报表 ✅

### 报表功能

**`modules/inventory_offline/reporter.py`**（新建）

`InventoryReporter` 类提供：

| 方法 | 功能 |
|------|------|
| `get_stock_snapshot()` | 当前库存快照，含位置分布/进货价/金额/最后出入库时间 |
| `get_movements()` | 出入库明细，支持日期/SKU/类型/订单号筛选 |

| `export_movements_to_excel()` | 出入库明细 Excel（IN绿/OUT红/ADJUST黄/RETURN蓝） |

**`inbound_service.py` 扩展**
- `get_all_stock()` 新增返回字段：`locations`、`last_inbound_at`、`last_outbound_at`、`cost_price`

### CLI

```bash
# 完整报表（快照 + 明细）
python main.py inventory offline report

# 只导出快照
python main.py inventory offline report --type snapshot -o snapshot.xlsx

# 只导出明细（指定日期范围）
python main.py inventory offline report --type movements \
  --start-date 2026-04-01 --end-date 2026-04-30 \
  --sku TEST-SKU-001
```

## 技术问题与修复

### Bug 1：`Inventory.cost_price` 不存在
- **原因**：测试 fixture 试图给 `Inventory` 赋 `cost_price`，但 `Inventory` 模型无此字段
- **修复**：去掉 fixture 中的 `cost_price=`，`get_movements()` 返回 `unit_cost=None`

### Bug 2：`DetachedInstanceError`
- **原因**：`get_movements()` 在 `with get_session()` 块外访问 `r.occurred_at` 等属性
- **修复**：在 `with` 块内提前构建 `MovementItem` 列表再返回

### Bug 3：`get_trend` daily 字典始终为空
- **原因**：`r.type.value` 是小写 `"in"`，但比较用的是 `"IN"`
- **修复**：统一改为小写比较

### Bug 4：时区 naive vs aware 比较
- **原因**：`datetime.now(timezone.utc)` 生成 aware datetime，与 SQLite naive `occurred_at` 不兼容
- **修复**：改用 `datetime.now()` naive datetime

### Bug 5：`main.py` `p_stk_list` 变量误删
- **原因**：之前修改 main.py 时把 `p_stk_list =` 这一行合并掉了
- **修复**：恢复 `p_stk_list =` 变量声明
