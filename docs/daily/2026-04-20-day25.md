## T1 完成：unit_cost / total_cost 回填 ✅

### 实现

`reporter.py` — `get_movements()` 成本填充逻辑：

| 变动类型 | unit_cost 来源 | total_cost |
|----------|---------------|------------|
| IN / RETURN | `InboundReceiptItem.cost_price`（通过 `related_order`=receipt_no join） | `unit_cost × quantity` |
| OUT / ADJUST | `Product.cost_price`（fallback） | `unit_cost × quantity` |

**join 查询优化（N+1 → 1 query）：**

```python
# 优化前：N+1
items = sess.query(InboundReceiptItem).join(...).filter(...).all()
for item in items:
    receipt = sess.query(InboundReceipt).filter(...).first()  # 每条多一次查询

# 优化后：1 query
rows_cost = (
    sess.query(InboundReceiptItem, InboundReceipt.receipt_no)
    .join(InboundReceipt, InboundReceipt.id == InboundReceiptItem.receipt_id)
    .filter(InboundReceipt.receipt_no.in_(receipt_nos))
    .all()
)
for item, receipt_no in rows_cost:
    cost_by_receipt_sku[(receipt_no, item.sku)] = item.cost_price
```

当前数据量小无感，但历史订单迁移（Phase 5 后期可能上万条）时 N+1 会变慢瓶颈，已顺手修复。
