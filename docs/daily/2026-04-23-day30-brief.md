# Day 30 Brief v0 — 按月 / 按天分解报表 (Finance Breakdown)

**前置状态**: Day 29 收口后基线，Finance Dashboard 已可用，JPY 本位语义已固化。

## 目标
在 Day 29 总览看板之上，新增“时间分解报表”，支持：
- 按月汇总
- 按天汇总
- 与 Day 29 完全一致的财务语义
- 纯 CLI 文本输出

## 决策记录
1. **继续使用 JPY 本位**，所有聚合只基于 `Transaction.amount_jpy`。
2. **复用 Day 29 语义**，不重新发明公式。
3. **只读服务**，不写库，不加 migration。
4. **CLI 入口继续挂 `main.py`**，不新建独立 finance cli。
5. **输出优先可读性**，不做 Excel，不做图表。

## 核心语义
### 1. 聚合口径必须与 Day 29 保持一致
- `total_revenue_jpy = Σ amount_jpy where type IN (SALE, SHIPPING, REFUND)`
- `total_cost_jpy = Σ total_cost where type = SALE`
- `total_fee_jpy = Σ |amount_jpy| where type = FEE`
- `gross_profit_jpy = total_revenue_jpy - total_cost_jpy - total_fee_jpy`
- `gross_margin = gross_profit_jpy / total_revenue_jpy`
- `amount_jpy IS NULL` 不参与金额聚合，但需要统计 `uncovered_count`
- `ADJUSTMENT` 排除所有聚合

### 2. 分解维度
- `group-by month`: `YYYY-MM`
- `group-by day`: `YYYY-MM-DD`

### 3. 结果字段
每一行至少返回：
- period
- revenue_jpy
- cost_jpy
- fee_jpy
- gross_profit_jpy
- gross_margin
- orders
- uncovered_transactions
- coverage_ratio

## CLI 草案
```bash
python main.py finance breakdown --group-by month --period all
python main.py finance breakdown --group-by day --period last-30-days
python main.py finance breakdown --group-by day --from 2026-04-01 --to 2026-05-01 --period custom
```

参数建议：
- `--group-by {day,month}` 必填
- `--period {all,this-month,this-week,last-7-days,last-30-days,custom}` 默认 `this-month`
- `--from YYYY-MM-DD`
- `--to YYYY-MM-DD`（半开上界）
- custom 模式缺参数时报友好错误
